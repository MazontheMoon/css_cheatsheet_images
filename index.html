The road so far...

Continuing with my self taught frontend development coding journey I have created a series of CSS Cheat sheets. I have gone from how to select an element and position it, to how to style text based content and now I am moving on to the more visual styling components like images.

Everything covered so far has looked at using images for backgrounds or bullet points so this cheat sheet is covering how to manipulate images when used as multimedia elements.

The focus is using filters to apply graphical effects (which can also be used on backgrounds and borders), or by clipping the image.

Filters...

filter:function(value);

Filters can be combined:

img {
filter: contrast (175%) brightness (75%);
}
  • url
  • blur
  • brightness
  • contrast
  • drop-shadow
  • grayscale
  • hue-rotate
  • invert
  • saturate
  • opacity
  • sepia

url()

...is used to reference a SVG filter

.img_box{
filter: url(resource.svg#elementID);
}

blur()

...applies Gaussian blur (does not accept percentages as value)

img {
filter: blur(5px);
}

brightness()

...as it sounds (0% = black and 100% is unchanged â€“ can also go above 100%!)

img {
filter: brightness(0.5);
}

contrast()

...as it sounds (0% = gray and 100% is unchanged â€“ and can go above 100%!)

img {
filter: contrast(200%);
}

drop-shadow()

...applies a drop-shadow effect to the input image.

img {
/*filter: drop-shadow(offset-x offset-y blur-radius color);*/
filter: drop-shadow(5px 5px 2px black);
}

grayscale()

...converts image to greyscale (0% is unchanged and 100% is completely greyscale)

img {
filter: grayscale(50%);
}

hue-rotate()

...applies hue-rotation to the image (different colour on colour wheel)

img {
filter: hue-rotate(90deg);
}

invert()

...inverts the colours (100% is opposite colour on wheel and 0% is unchanged)

img {
filter: invert(33%);
}

saturate()

...saturates the input image (0% is unsaturated and 100% is unchanged and can over-saturate by going over 100%)

img {
filter: saturate(200%);
}

opacity()

...changes â€œsee-throughnessâ€œ (0% is completely transparent and 100% is unchanged)

img {
filter: opacity(50%);
}

sepia()

...converts image to sepia colouration (0% is unchanged and goes to 100% )

img {
filter: sepia(20%);
}

Clipping...

The property creates a clipping region that sets what part of an element should be shown (parts that are inside the region are shown, while those outside are hidden)

img {
clip-path: function(value);
}
  • inset
  • circle
  • ellipse
  • polygon
  • path
  • and by "boxes"...

inset()

img {
clip-path: inset(100px 50px);
}

circle()

img {
clip-path: circle(50px at 0 100px);
}

ellipse()

img {
clip-path: ellipse(50px 60px at 0 10% 20%);
}

polygon()

img {
clip-path: polygon(50% 0%, 100% 50%, 50% 100%, 0% 50%);
}

Use this clippytool

path()

img {
clip-path: path('M0.5,1 C0.5,1,0,0.7,0,0.3 A0.25,0.25,1,1,1,0.5,0.3 A0.25,0.25,1,1,1,1,0.3 C1,0.7,0.5,1,0.5,1 Z');
}

Boxes...

img {
clip-path: padding-box;
}

There are combos too such as...

img {
clip-path: padding-box circle(50px at 0 100px);
}

Tips and Tricks...

There can also be a situation where the image doesn't load and while there is always the alt text there is also a more stylish way of handling images with broken links

img:after {
/*in case of image not found*/
display: inherit;
position: absolute;
top: 0;
left: 0;
width: 100%;
padding: 5px;
border: 1px dashed slategrey;
background-color: lightgrey;
z-index: 1;
font-family: verdana;
font-size: 0.8em;
content: "" attr(alt) " was not found";
}

And don't forget about the inherit, initial and unset values!
